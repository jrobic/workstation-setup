# Commit signing — per-machine GPG keys

Goal: sign commits, verified on GitHub **and** GitLab, with one key **per machine**.

## Model

- **One GPG signing key per machine** (like the SSH key). The private key lives in `~/.gnupg`, **never versioned**.
- `user.signingkey` = the **public fingerprint** of the key. This is **not a secret** (it's a public identifier), but it's **per-machine** → we isolate it in `~/.gitconfig.local`, not versioned.
- The versioned `~/.gitconfig` (added via `chezmoi add --template`) includes the local one:

```ini
# ~/.gitconfig (versioned, chezmoi)
[include]
	path = ~/.gitconfig.local      # signingkey + gpgsign, written by gpg role
[user]
	name = Jonathan Robic
	email = hello@jonathanrobic.fr
[gpg]
	program = gpg
```

```ini
# ~/.gitconfig.local (per-machine, generated by ansible gpg role — NOT versioned)
[user]
	signingkey = <fingerprint>
[commit]
	gpgsign = true
[tag]
	gpgsign = true
```

## Automated by the `gpg` role

`ansible/roles/gpg` does the following at bootstrap:

1. Generate an ed25519 GPG signing key for `hello@jonathanrobic.fr` if absent.
2. Write `~/.gitconfig.local` (signingkey + gpgsign).
3. Upload the public key: GitHub `gh gpg-key add`, GitLab `glab api --method POST /user/gpg_keys` (no `glab gpg-key` subcommand).

## Manual fallback

```bash
# Generate (without passphrase)
gpg --batch --pinentry-mode loopback --passphrase '' \
    --quick-generate-key "Jonathan Robic <hello@jonathanrobic.fr>" ed25519 sign never

# Fingerprint
FPR=$(gpg --list-secret-keys --with-colons hello@jonathanrobic.fr | awk -F: '/^fpr:/{print $10; exit}')

# Local git config
git config --file ~/.gitconfig.local user.signingkey "$FPR"
git config --file ~/.gitconfig.local commit.gpgsign true

# Upload
gpg --armor --export "$FPR" > /tmp/pub.asc
gh gpg-key add /tmp/pub.asc
glab api --method POST /user/gpg_keys --field key="$(cat /tmp/pub.asc)"
```

## Passphrase tradeoff

The role generates the key **without a passphrase** → frictionless signing, but the private key is usable as-is if `~/.gnupg` leaks (personal machine, risk accepted).

Safer variant: generate with passphrase, store it in a Dashlane note, and pre-load it in `gpg-agent` at login (`gpg-preset-passphrase`). Implement if needed.

## Important: GPG email = commit email

The GPG key email **must** match `user.email` git **and** be a verified email on GitHub/GitLab, otherwise the signature shows "Unverified". Verify that `hello@jonathanrobic.fr` is confirmed on both accounts.

## Alternative not chosen: SSH signing

Git can also sign with the **SSH** key already generated (`gpg.format = ssh`, `user.signingkey = ~/.ssh/id_ed25519.pub`) — zero extra keys. Rejected here in favor of GPG (explicit choice). Reconsider if GPG management becomes burdensome.
